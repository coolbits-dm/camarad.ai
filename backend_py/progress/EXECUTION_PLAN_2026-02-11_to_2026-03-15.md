# Execution Plan (2026-02-11 -> 2026-03-15)

Scop: plan de executie concret pe 30 de zile, cu gates clare.

## Phase A (Zilele 1-3): M2.A Google Ads real via Coolbits

Owner: `codex`

### Deliverables
- Wire Google Ads endpoint-uri la Coolbits:
  - `/api/connectors/google-ads/accounts`
  - `/api/connectors/google-ads/campaigns`
  - `/api/connectors/google-ads/keywords`
  - `/api/connectors/google-ads/metrics`
- Feature flag operational:
  - `COOLBITS_GATEWAY_ENABLED=1` activeaza gateway
  - `0` mentine mock fallback
- Error handling:
  - timeout clar
  - fallback automat pe mock in caz de fail gateway
  - raspuns standardizat cu camp `source: coolbits|mock`

### Acceptance Gate A
- endpoint-urile de mai sus raspund corect pe date reale cand gateway e on
- UI connectors Google Ads functioneaza fara crash
- cand gateway cade, UI continua pe mock (degraded mode)

## Phase B (Zilele 4-7): M2.B GA4 + GSC/GTM bridge

Owner: `codex`

### Deliverables
- Mapeaza endpoint-uri GA4 critice la Coolbits:
  - `properties`, `overview`, `timeseries`
- Introdu endpoint health pentru gateway:
  - `/api/connectors/gateway/health`
- Adauga cache scurt (30-60s) pentru dashboard-heavy routes

### Acceptance Gate B
- dashboard GA4 incarcat cu date reale pe minim un cont
- latenta endpoint-uri critice sub prag stabilit intern
- fallback mock activ la indisponibilitate upstream

## Phase C (Saptamana 2): M3 Client data safety audit

Owner: `codex + cblm`

### Deliverables
- Lista endpoint-uri critice + matrix cu suport `X-Client-ID`
- test suite automat:
  - user A/client A nu vede date user A/client B
  - user B nu vede nimic de la user A
- remedieri unde lipseste scoping consistent

### Acceptance Gate C
- zero leakage in test suite pe chat/flows/connectors/history/settings
- endpoint-urile critice au test coverage minima

## Dependencies and blockers

- Pentru M2 real data:
  - acces operational la Coolbits endpoint-uri pe server
  - configurare corecta `COOLBITS_URL`, `COOLBITS_GATEWAY_EMAIL`, `COOLBITS_WORKSPACE_ID`
- Pentru backup remote:
  - credentials VPS secundar sau S3 endpoint (amanat)

## Change control

- Fiecare faza are rollback:
  - feature flag off pentru gateway
  - revert endpoint mapping la mock-only
  - Nginx/PM2 config snapshots pastrate
- Orice incident major actualizeaza:
  - `progress/STATUS_YYYY-MM-DD.md`
  - `progress/NEXT_STEPS.md`
